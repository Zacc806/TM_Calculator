# Казахский перевод интерфейса — лист для подписи носителя

**ТЗ Задача 1.** Цель: носитель казахского языка (РК) подтверждает, что перевод звучит естественно и
соответствует банковской терминологии (Отбасы, Halyk, Kaspi), либо вносит правки в колонку «KZ».

Версия от 2026-06-08. Прошёл двойную AI-ревизию (предложение + независимая проверка нативным
банковским редактором); 14 строк из 75 улучшено под банковскую терминологию
и грамматику. **Колонка «KZ» — то, что сейчас в продакшене.** Носителю: отметьте/поправьте строки,
где нужно иначе. Финальные правки разработчик внесёт точечно в `src/i18n/translations.ts` (kk).

## Контент ипотечных программ (отдельно от UI)

Названия, описания, условия, требования, проекты и «актуальность» 17 программ локализованы в данных
(`src/data/programs.seed.json` → поле `i18n.kk`, и на сервере `programs.json`), переведены отдельным
воркфлоу с сохранением всех цифр. Их также стоит дать носителю на сверку (длинный юридический текст);
правится через админку `/admin` или в `i18n.kk` программ.

## Как внести правки носителя

`src/i18n/translations.ts`, блок `kk: { ... }` — найти ключ, заменить строку. Затем
`npm test` (тест паритета ключей) → `npm run build` → деплой.
